2. **文字ラベルは英語**、本文キャプションで日本語 — 英語版改訂時に再制作を回避（設計ルール 7）
3. **SVG が第一選択肢** — テキスト編集可能、PDF/HTML 共通、テーマ変数対応

6. **配置**：`images/` 配下、命名規則は `images/{chapter}-{slug}.svg`（例：`images/c11-three-processes.svg`）

## 実装メモ

### SVG ファイルのテンプレート（共通）

- viewBox は 16:10（800x500）を基本
- ストロークは 2px、ラベルは 14-16px sans-serif
- カラーパレット：本書テーマと整合（cover.svg 参照）
  - アクセント色：#ffd97a（黄、ハイライト）
  - 主線：#1f4e79（濃紺）
  - 副線：#a8c5e0（薄青）
  - 強調：#c43f6b（マゼンタ）
- 文字は英語、フォント指定なし（システム sans-serif）

### matplotlib プロットの共通設定

```python
matplotlib.rcParams.update({
    "font.family": "DejaVu Sans",
    "font.size": 11,
    "axes.labelsize": 12,
    "axes.titlesize": 13,
    "figure.dpi": 200,
    "savefig.bbox": "tight",
    "savefig.pad_inches": 0.1,
})
```

これを `code/_style.py` に集約して各スクリプトから import する。

### 図キャプションの日本語化

本文 .qmd 側で `![日本語キャプション](images/file.svg)` のように書けば、SVG 内部は英語のままで本文側は日本語表示になる。英訳時に SVG を作り直さなくて済む。

## 進行管理

- **GitHub Issues**：図ごとに 1 Issue を立てて、ラベル `figure`, `v0.2`〜`v1.0` で管理
- **GitHub Projects**：図の進捗ボード（Backlog → In Progress → Review → Done）
- **PR 単位**：1 枚 1 PR が原則。中心地図の 4 変種など派生は 1 PR にまとめてもOK

各図の Issue タイトルテンプレ：

```
[Figure] §X.Y - 図の説明（v0.X target）
```

例：`[Figure] §11.2 - Three processes of two-level system (v0.2 target)`
